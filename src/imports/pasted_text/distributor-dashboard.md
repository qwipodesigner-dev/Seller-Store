Design a modern "Dashboard" page for a B2B Seller Application (Distributor Dashboard).

Objective:
Create a clean, high-impact dashboard that focuses on key numbers and actionable insights.
Avoid traditional cluttered dashboards with too many charts.

---

### 🧩 Page Structure

1. Header (Filters)
2. Key Metrics (Number Cards)
3. Insights Section (Smart Highlights)
4. Quick Actions (Optional)

---

### 🧠 1. Header

- Title: "Dashboard"
- Date Filter (Last 7 / 30 / Custom)
- Channel Filter (ONDC / All)

---

### 🔢 2. Key Metrics (Top Priority)

Display 6–8 clean number cards:

- Total Orders
- Total Revenue
- Avg Order Value
- Active SKUs
- Low Stock SKUs
- Orders in Progress
- Completed Orders
- Cancelled Orders

Design:
- Card-based layout (2 rows)
- Large bold numbers
- Small label + % change (optional)
- No graphs inside cards

---

### 💡 3. Insights Section (VERY IMPORTANT)

Instead of charts, show smart insights:

Examples:
- "⚠️ 12 SKUs are low on stock"
- "🔥 Top product: XYZ with highest sales"
- "📉 Orders dropped by 8% vs last week"
- "🚀 3 products driving 60% revenue"

Design:
- Highlight cards or list
- Use icons + color indicators
- Keep it readable in 5 seconds

---

### 📦 4. Optional Mini Sections (Keep Minimal)

A. Top Products (Simple list)
- Product Name
- Orders / Revenue

B. Recent Orders (Last 5)
- Order ID
- Status
- Value

---

### 🎨 Design Guidelines

- Clean SaaS UI (Stripe / Razorpay style)
- No heavy charts
- Use whitespace generously
- Soft shadows, rounded cards
- Consistent spacing

---

### ⚡ UX Principles

- Show only what matters
- Quick glance = full understanding
- Prioritize actions over analytics

---

### 🚫 Avoid

- No complex graphs
- No overcrowded widgets
- No deep analytics here

---

### 🎯 Goal

User should:
- Understand business health in 5 seconds
- Identify issues (low stock, drops)
- Take action quickly
